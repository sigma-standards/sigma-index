§ 0

Definition of 100 Percent

100 percent does not mean every paywalled full text is copied. It means SIGMA has the best lawful open metadata and source references for every major standards family, and full public-text harvesting where licensing permits.

0.1 Completion Criteria

  • ✓
    Domain coverage: all 40 canonical domains have source-backed records beyond seed placeholders.
  • ✓
    Layer coverage: global, regional, national, sectoral, treaty, classification, technical, governance, and implementation layers are represented.
  • ✓
    Source coverage: every active source family has an entry in data/reference/source_registry.csv.
  • ✓
    Task coverage: every domain has planned, active, or completed tasks in data/reference/research_tasks.csv.
  • ✓
    Processed data: every promoted source writes schema-valid rows into data/processed/.
  • ✓
    Staging discipline: broad harvesters write to data/staging/ until curator filters are proven.
  • ✓
    Relationship graph: issuer, successor, amendment, adoption, domain, treaty, and implementation relationships are captured in data/relationships/.
  • ✓
    Quality gates: duplicate IDs, missing required fields, malformed URLs, broken local docs, and invalid relationships are checked before release.
  • ✓
    Publication: GitHub Pages renders the landing page, downloads, domain coverage, source registry, task coverage, quality reports, and project documentation as HTML.
  • ✓
    Release governance: v1.0 has release notes, a Git tag, persistent archival copy, and contribution workflow.
